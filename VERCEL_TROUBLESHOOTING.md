# Настройка для Vercel

## Проблема: Белый экран

Если видите белый экран на Vercel, скорее всего проблема в настройке API URL.

## Решение

### Вариант 1: Backend на отдельном домене

1. Задеплойте backend (например, на Railway, Render, или другой хостинг)
2. В настройках Vercel проекта добавьте переменную окружения:
   ```
   VITE_API_URL=https://your-backend-url.com
   ```
3. Пересоберите проект на Vercel

### Вариант 2: Backend на том же домене (через Vercel Serverless Functions)

Если backend тоже на Vercel, используйте относительные пути (уже настроено в коде).

### Вариант 3: Проверка ошибок в консоли

1. Откройте DevTools (F12)
2. Перейдите на вкладку Console
3. Посмотрите какие ошибки там есть
4. Обычно это:
   - `Failed to fetch` - проблема с API URL
   - `Cannot read property...` - ошибка в коде
   - `Module not found` - проблема со сборкой

## Быстрая проверка

Откройте консоль браузера и проверьте:
- Есть ли ошибки JavaScript?
- Какой API URL используется? (должен быть в логах)
- Работает ли API? (попробуйте открыть `/api/channels` напрямую)

## Текущая конфигурация

В `frontend/src/lib/api.ts`:
- В продакшене: использует `VITE_API_URL` или относительный путь
- В разработке: использует `http://localhost:8000`

Если `VITE_API_URL` не установлен в продакшене, API будет использовать относительные пути (тот же домен).
